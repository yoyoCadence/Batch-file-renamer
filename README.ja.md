# Batch File Renamer PWA

[繁體中文](README.md) | [English](README.en.md) | **日本語**

プレビューしてから実行する、ローカルファイル向けの一括リネームツールです。  
旧 PowerShell ユーティリティを静的 PWA として作り直し、大量のファイル名変更という危険な操作を、実行前に確認できる流れへ変えています。

## すぐ使う

Live App: https://yoyocadence.github.io/Batch-file-renamer/

こんな場面に向いています：

- 写真、PDF、スキャン、帳票、エクスポートファイルをまとめて整理する。
- セグメント、文字範囲、固定値、連番、削除、行ごとのリストで名前を作る。
- 実ファイルを変更する前に、すべての結果を表で確認する。
- 1 つのテンプレートファイルを出力フォルダーへ複製し、それぞれ別名で保存する。

## 安心して使える理由

一括リネームで怖いのは、クリックした瞬間に何が起きるか見えないことです。このアプリは重要な判断を画面上に出します：

- **先にプレビュー**：すべての変更後ファイル名を表に表示し、手動編集もできます。
- **OK 行だけ実行**：警告、エラー、重複、変更なしの行は実行対象から分けて表示します。
- **モードが明確**：名前変更は元の場所でファイル名を変更します。コピーはテンプレートを出力先へ複製してから名前を変えます。
- **パスを確認できる**：通常は現在のファイル名だけ表示し、ホバーまたは切り替えでフルパスを表示できます。
- **システムファイルを除外**：`desktop.ini`、`Thumbs.db` などの一般的なシステムファイルや一時ファイルを読み込み時に除外します。

## 主な機能

- Chromium の File System Access API を使ったローカルフォルダーの直接リネーム。
- テンプレートファイル、出力フォルダー、出力数を指定できるコピー作成モード。
- セグメント、文字範囲、固定値、リスト、昇順、降順、削除に対応したルールビルダー。
- 行ステータス、変更後ファイル名編集、出力先ツール、OK 行のみ実行を備えたプレビュー表。
- CSV インポート/エクスポートによる手動確認や表計算ソフト連携。
- UI 言語：繁體中文、简体中文、English、日本語。
- 20 種以上の高精細背景テンプレートと 20 種以上のカラーテーマ。
- 高精細マスコット、シンプル図形版、漫画風ふきだし、ドラッグ反応、自然な移動モードを持つペット助手。

## ペットシステム

現在の新しい移動システムの試作は **どこでもドア書類修理屋** です。画面をただ漂うのではなく、自然な移動をします：

- 床やパネルの端を歩きます。
- 別の場所へ移動するとき、どこでもドア、竹コプター、ロープの演出を使います。
- ドラッグ中は驚いた表情の held ポーズになります。
- セリフは、一方的な質問ではなく、短い付き添いの言葉として書き直しています。

従来の自由に漂う移動も、設定から選べます。

## 技術詳細

バックエンド不要の静的 PWA です：

- `pwa/index.html`：メイン UI 構造。
- `pwa/assets/app.js`：アプリ状態、ファイル選択、プレビュー、実行、設定、ペット挙動。
- `pwa/assets/rules.js`：ファイル名ルールエンジン、CSV 変換、プレビュー行の検証。
- `pwa/assets/settings.js`：言語、テンプレート、テーマ、ペット、設定正規化、翻訳辞書。
- `pwa/assets/style.css`：レスポンシブレイアウト、テーマ変数、背景テンプレート、ペットアニメーション。
- `pwa/service-worker.js`：PWA の runtime cache。素材更新時は cache version を更新し、アプリ側で新バージョンへの切り替えを案内します。
- `tests/*.test.mjs`：Node test runner によるルール、i18n、静的素材、service worker cache、UI hook のテスト。

ファイル操作の方針：

- 直接リネーム/コピー実行は File System Access API に依存するため、Chromium 系ブラウザーが最適です。
- この API がないブラウザーでも、ルール作成、プレビュー、CSV インポート、CSV エクスポートは使えます。
- ブラウザーのフォルダー選択画面は権限ダイアログであり、ファイル内容ではなくフォルダーだけを表示する場合があります。個別ファイルを選ぶ場合は **Select files** を使ってください。

## ローカル実行

```powershell
py -m http.server 4173 --directory pwa
```

開く：

```text
http://127.0.0.1:4173/index.html
```

## 検証

```powershell
npm run test
npm run build
```

現在のテスト対象：

- ルールエンジンとプレビュー行の検証。
- 4 言語間の翻訳キー一致。
- 20 種以上の背景テンプレートと 20 種以上のテーマ。
- 高精細背景素材のサイズしきい値。
- ペット素材、自然移動 hook、service worker cache エントリ。
- テキストファイル内の未解決 merge marker チェック。
